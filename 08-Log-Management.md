# Chapter 8: Log Management

## Overview
Log management is one of the most important responsibilities of a Linux system administrator. Logs provide detailed records of operating system activity, user authentication, application events and system services. They are essential for troubleshooting, auditing, performance monitoring and cybersecurity investigations. Ubuntu stores logs in two primary ways:
- Traditional log files located in `/var/log`
- The **systemd journal**, which is accessed using the `journalctl` command

In this chapter, I explored the Linux logging system, examined authentication logs, viewed service logs using `journalctl` and reviewed how Ubuntu automatically manages log files using Logrotate.

# Exploring System Log Files

Ubuntu stores most system logs inside the `/var/log` directory.

To explore the available logs, I used:

```bash
ls -lh /var/log
```
Some important log files include:

| Log File | Description |
|----------|-------------|
| auth.log | Authentication and SSH login events |
| syslog | General system activity |
| kern.log | Linux kernel messages |
| dpkg.log | Package installation history |
| boot.log | Boot process information |
| lastlog | Last login information for users |

To review recent entries from authentication and system logs I used following commands. These commands display the most recent log entries without opening the entire log file.

```bash
sudo tail -20 /var/log/auth.log
```

```bash
sudo tail -10 /var/log/syslog
```

# Viewing Logs with journalctl

Modern Ubuntu systems use **systemd-journald** to collect and manage logs generated by the operating system and services. journal provides a centralized logging system that allows administrators to efficiently search and filter log entries.

To view logs generated by the SSH service I used:

```bash
sudo journalctl -u ssh
```
This command displays all journal entries related to the SSH service, including:

- Service startup
- Service shutdown
- Login attempts
- Authentication events
- SSH daemon activity

Other useful `journalctl` commands include:

Display today's logs:

```bash
sudo journalctl --since today
```

Display the most recent 50 log entries:

```bash
sudo journalctl -n 50
```

Display logs from the current boot:

```bash
sudo journalctl -b
```

These commands are useful for troubleshooting and investigating recent system activity.

# Reviewing Authentication Logs

Authentication logs provide detailed information about user login attempts and SSH authentication events.

To display successful SSH logins, I used:

```bash
sudo grep "Accepted" /var/log/auth.log
```

To identify failed login attempts, I used:

```bash
sudo grep "Failed" /var/log/auth.log
```

These commands allow administrators to quickly identify:

- Successful SSH logins
- Failed authentication attempts
- Brute-force attacks
- Invalid user login attempts

Authentication logs are commonly reviewed during security monitoring and forensic investigations.

# Understanding Log Rotation

Without maintenance log files would continuously grow and eventually consume significant disk space. Ubuntu automatically manages log files using **Logrotate** which archives old logs and creates new ones according to predefined policies.
The main Logrotate configuration file can be viewed using:

```bash
cat /etc/logrotate.conf
```

Package specific log rotation policies are stored in:

```bash
ls /etc/logrotate.d
```

The default configuration demonstrates several important features:

- Weekly log rotation
- Keeping four weeks of archived logs
- Automatic creation of new log files
- Optional log compression
- Separate rotation policies for installed services

Log rotation helps maintain system performance while preserving historical log information for auditing and troubleshooting.
